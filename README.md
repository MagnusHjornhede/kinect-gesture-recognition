# Kinect Gesture Recognition

Gesture classification using skeleton tracking data from a **Microsoft Kinect sensor**.

The project explores how classical machine learning models perform on **high-dimensional skeleton-based gesture features**, and investigates how preprocessing techniques such as **PCA** affect model performance.

<br>

# Pipeline

skeleton tracking → feature extraction → preprocessing → ML classifier → evaluation

<br>

# Dataset

Kinect skeletal tracking data.

- 20 joints tracked per frame
- 242 numeric features derived from skeleton coordinates and motion attributes

Labels:
- gesture name (categorical)
- gesture ID (numeric)

Data format:

- CSV files without headers
- Headers added in code as `feature_1 ... feature_242`

Preprocessing:

- Missing values imputed using column means
- Z-score standardization applied to all features

The dataset contains **multiple gesture classes**, with some class imbalance between training and test sets.

<br>

# Experiment Overview

This repository contains a set of **machine learning experiments on Kinect skeleton gesture data**.

The experiments are organized in **three stages**, all using the same dataset (242 skeleton-derived numeric features + labels).

### Stage 1 — Data Preparation

- missing-data audit
- imputation
- scaling
- exploratory analysis
- visualization

### Stage 2 — Baseline ML Models

Training and evaluation of multiple classical ML models.

### Stage 3 — PCA vs Raw Features

Benchmarking the effect of **PCA (95% variance)** versus using the raw feature space.

The aim is to evaluate which models handle this type of gesture data best and to understand how preprocessing choices (like PCA) influence performance.

# Headline results

Key observations from the experiments:

- **ExtraTrees and RandomForest** perform best overall
- **Linear SVM** becomes highly competitive after PCA
- PCA (95%) improves linear models but reduces performance of tree ensembles

General pattern:

- without PCA → tree ensembles dominate
- with PCA → linear SVM achieves highest accuracy 

<br>

# Model Insights

### ExtraTrees / RandomForest

Perform strongly because they handle **high-dimensional tabular data with redundant features**, selecting useful splits and averaging out noise.

### SVM (linear / RBF)

Effective on standardized numeric features.

- RBF captures nonlinear relationships  
- linear SVM benefits from PCA reducing feature redundancy

### GradientBoosting / AdaBoost

Show weaker performance without careful tuning, likely due to sensitivity to noise and hyperparameters.

### Bagging

Improves stability of weaker learners (e.g., linear SVM), especially after PCA.

### PCA Effects

PCA reduces collinearity and redundancy in the feature space.

- improves linear models
- often harms tree ensembles that rely on original feature axes

<br>

# Conclusions

From these experiments we observe:

1. Tree ensembles are the most robust baseline for Kinect skeleton gesture data without heavy preprocessing.
2. PCA shifts the advantage toward linear models by compressing the feature space.
3. Model performance strongly depends on the preprocessing strategy.
4. No single model dominates across all conditions.

<br>

# Recommended Baselines

For practical gesture recognition pipelines:

- **RandomForest / ExtraTrees**  
  Strong default for raw high-dimensional feature spaces.

- **Linear SVM + PCA**  
  Effective when dimensionality reduction and simpler models are preferred.
